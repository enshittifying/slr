Automated Editorial Pipeline

The system implements a three-stage automated pipeline:

  1. Stage 1: Source Pull Machine (sp_machine) - Automatically finds and downloads source materials from legal databases, websites, and academic repositories.
  2. Stage 2: R1 Machine (Preparation) - Prepares sources by cleaning PDFs, performing metadata redboxing, and filing sources to Google Drive.
  3. Stage 3: R2 Pipeline (Validation) - LLM-powered validation of citation formatting, proposition support checking, and quote verification.

Key Features

Automated Source Discovery

Connects to HeinOnline, Westlaw, Lexis, and other databases to automatically find format-preserving PDFs.

AI-Powered Validation

Uses large language models to check Bluebook compliance and verify proposition support.

Google Workspace Integration

Native integration with Google Sheets, Forms, Drive, and Calendar for seamless workflow management.

Per-Article Processing

Each article (e.g., 78.6, 79.1) undergoes its own complete SP → R1 → R2 pipeline.
